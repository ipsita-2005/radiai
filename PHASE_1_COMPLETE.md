## 🔧 Technical Implementation

### Stage 1: Strict File Extension & MIME Check

**Routing Rules:**
```python
.csv → ECG (electrocardiogram data)
.dcm/.dicom → CT (DICOM format)
.jpg/.jpeg/.png → Image pipeline (needs further analysis)
```

**Validation:**
- MIME type checking
- Extension validation
- Unsupported format rejection with clear error

### Stage 2: Enhanced Pixel Statistics

**Features Analyzed:**
```python
✓ Mean pixel intensity
✓ Standard deviation
✓ Aspect ratio (width/height)
✓ Histogram distribution
✓ Edge density (Sobel operator)
✓ Texture features
```

**Classification Heuristics:**

**Brain MRI Detection:**
- Square-ish images (0.8 ≤ AR ≤ 1.2)
- Mid-range intensity (60-140)
- High contrast variation (std > 40)

**Chest X-Ray Detection:**
- Rectangular images (AR < 0.7 or AR > 1.5)
- High brightness (>150 mean intensity)
- Bone structure patterns

**CT Scan Detection:**
- Cross-sectional view patterns
- Specific HU ranges (100-200 intensity)
- Higher edge density (>20)

### Stage 3: Error Handling

**When Detection Fails:**
```python
HTTP 400 Error with message:
"Unable to automatically determine imaging modality. 
Please manually select the modality type:
- Brain MRI: Upload brain scan images
- Chest CT: Upload chest CT scans  
- X-Ray: Upload X-Ray images
- ECG: Upload ECG signal data (CSV format)"
```
